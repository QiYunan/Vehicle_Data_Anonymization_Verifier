# 匿名化与检测要求（GB/T 44464-2024 计算部分梳理）

> 本文件梳理 **GB/T 44464-2024** 中与"计算/判定"相关的全部要求，作为脚本三（正式检测计算引擎）及自检评测的需求依据。
> 按项目**两个阶段**组织：**阶段一·自检（标注精度）** 与 **阶段二·正式检测**。
> 最后更新：2026-06-23

---

## 0. 两套"最小边长"阈值（最易混，先厘清）

国标存在**两套互不相同**的最小边长阈值，用途完全不同，**互不干扰**：

| 阈值来源 | 人脸 | 车牌 | 用途 |
|---|---|---|---|
| **§5.6.2.1 匿名化对象定义** | 最小边长 ≥ **32px** | 最小边长 ≥ **16px** | 判"**该不该**打码"——应检数清点、检出率（自检项5、项7 + 阶段二） |
| **B.2.4.2 标注精度阈值** | 最小边长 ≥ **27px** | **11–20px / >20px** 分档 | 判"标注**准不准**"——自检项1~4 |

> ⚠️ 现有 `gbt_step5_evaluator.py` 里用的是人脸 32px、车牌 16px、小/大牌界 32px，**与 B.2.4.2 不符**（应为 27px、车牌 11–20 / >20）。后续修脚本五时需按本表纠正。

## 1. 关键术语定义（§3）

### 1.1 人脸目标（§3.5）
自然人**头部正面**：**眉毛最上端**至**颏底线**之间、**左耳到右耳（不含耳朵）**之间的部分。
- 不含：额头（眉毛以上）、头发/头顶、耳朵、脖子。

### 1.2 人脸边界框（§3.6）
覆盖人脸目标的**最小矩形或旋转矩形**。
- 标注要点：上沿贴眉毛最高点、下沿贴下巴底、左右沿贴到双耳内侧（排除耳朵）；取恰好罩住该区域的最小框（人脸倾斜时用旋转框更贴）。
- 不要按"整个头"框，否则最小边长/面积偏大，影响项1/3/4 比值与检出率。
- 配套示意：国标**图1**。

### 1.3 汽车号牌目标（§3.7）/ 边界框（§3.8）
- 号牌目标：安装于汽车、基材为金属的**正式机动车号牌**（不含喷涂放大号牌、纸质临时号牌）。
- 号牌边界框：号牌目标外沿组成的**最小矩形或旋转矩形**。

### 1.4 可见范围（§5.6.2.1.1 注1）
人脸边界框内**能直接观察到**的人脸区域；被遮挡部分从可见范围扣除。边界框大小不受影响。

### 1.5 遮盖率（§3.9）
人脸/号牌边界框内**已匿名化处理区域 ∩ 边界框** 的面积 ÷ 整个边界框面积。

### 1.6 检出率 / 误检率（§3.10 / §3.11）
- 检出率 = 正检数 / 应检数。（正检数 = 按要求已打码的目标数；应检数 = 按要求应打码的目标数）
- 误检率 = 误检数 / 检出目标数。（检出目标数 = 被标记并打码的目标数；误检数 = 其中不满足匿名化对象定义的）

## 2. 匿名化处理标准（§5.6.2.1，判"该不该打码"）

判定一个目标是否为**应检（须匿名化）对象**：

- **人脸目标**（§5.6.2.1.1，两条**同时**满足）：
  - 人脸边界框最小边长 ≥ **32px**；
  - 边界框内可见范围比值 > **50%**，且可见范围内**眼睛、鼻子或嘴清晰可见**。
  - 例外：广告牌、光滑表面倒影中的人脸**不算**对象。
- **汽车号牌目标**（§5.6.2.1.2，两条**同时**满足）：
  - 号牌边界框最小边长 ≥ **16px**；
  - 号牌**无遮挡**且**可识别全部数字及文字**。

## 阶段一 · 自检（标注精度，依据 B.2.4.2）

> 目的：证明"裁判"自身**标得准、数得准**。**本阶段不评判被测目标的匿名化效果**，只考核测量精度。
> 数据：未匿名化图 + 已匿名化图（人工标注真值）。**注：本项目自检数据为自拍约 100+ 张，非国标的各 500 张。**

### 未匿名化图（项1~5）

| 项 | 条件 | 计算 | 通过标准 |
|---|---|---|---|
| **1 人脸边界框精度** | 人脸框最小边长**真实值 ≥ 27px** | 每框「最小边长标注值 / 真实值」比值 | 比值落在 **[0.9, 1.1]** 的框数 **≥ 98%** |
| **2 小车牌边长误差** | 车牌最小边长**真实值 11 ≤ x ≤ 20px** | 「标注值 − 真实值」绝对值的**平均** | **≤ 1px** |
| **3 大车牌边长精度** | 车牌最小边长**真实值 > 20px** | 「标注值 / 真实值」比值的**平均** | 落在 **[0.9, 1.1]** |
| **4 人脸可见范围精度** | 可见范围 **< 100%** 且人脸框最小边长 **≥ 27px** | 「可见范围面积标注值 / 真实值」比值的**平均** | 落在 **[0.9, 1.1]** |
| **5 应检目标清点比** | 按 **§5.6.2.1** 标准数"应检目标" | 「算法识别应检数 / 人工实际应检数」（人脸、车牌） | 落在 **[0.99, 1.01]** |

### 已匿名化图（项6~7）

| 项 | 条件 | 计算 | 通过标准 |
|---|---|---|---|
| **6 遮盖率精度** | 遮盖率 **< 100%** 的匿名化对象 | 「标注遮盖率 − 真实遮盖率」绝对值的**平均** | **≤ 5%** |
| **7 正检/漏检清点比** | 按 §5.6.2.1 标注，分别数正检数、漏检数 | 「标注正检数/真实正检数」「标注漏检数/真实漏检数」各自比值 | 各项落在 **[0.98, 1.02]** |

> 项7 的正检/漏检判定线（来自 B.5.3）：满足 §5.6.2.1 且**遮盖率 ≥ 50% → 正检**；满足 §5.6.2.1 但**遮盖率 < 50% → 漏检**。

## 阶段二 · 正式检测（对主机厂数据出合规结论）

> 数据：**仅有主机厂发来的、已匿名化处理的视频**，**无未匿名化对照**。
> 流程：匿名化视频 → 抽帧（脚本一）→ 检测（脚本二）→ 计算各率（脚本三）。
> 核心判定：抽帧里**凡符合 §5.6.2.1 却没打好码（遮盖率 < 50% / 仍可识别）的人脸、号牌目标，一律计漏检**。
> 极端情况：主机厂图根本没打码，只要帧里有符合条件的目标，全计漏检 → 检出率极低 → 判不合格。

### 指标A · 匿名化检出率（§5.6.2.2.1，**强制**，公式 B.1/B.2）

对每个满足 §5.6.2.1 的真实目标，按遮盖率分类：
- **正检数** N_a：满足 §5.6.2.1 **且** 遮盖率 **≥ 50%**
- **漏检数** N_m：满足 §5.6.2.1 **但** 遮盖率 **< 50%**（含完全没打码）
- **应检数** = 正检数 + 漏检数

```
人脸检出率 R_df = N_af / (N_af + N_mf)
号牌检出率 R_dv = N_av / (N_av + N_mv)
```
- **通过：人脸检出率 ≥ 90% 且 号牌检出率 ≥ 90%。**

### 指标B · 漏检率（检出率的补数）

国标未单独命名，定义为：
```
漏检率 = 漏检数 / 应检数 = 漏检数 / (正检数 + 漏检数) = 1 − 检出率
```
- 分人脸、号牌各算：人脸漏检率 = N_mf/(N_af+N_mf)；号牌漏检率 = N_mv/(N_av+N_mv)。
- **通过：漏检率 ≤ 10%（等价于检出率 ≥ 90%）。**
- 分母为**应检数**（= 正检+漏检），**不含误检**，与误检率的分母（检出数）不同。

### 指标C · 匿名化误检率（§5.6.2.2.2，附录C，公式 C.1/C.2）

> 国标原文为"资料性 + 宜"，**本项目（视作强制）**。

针对主机厂系统**标记并打码**的每个匿名化对象：
- **误检数** N_F：被标为人脸/号牌打码，但**与任一真实人脸/号牌无交集**。
  - 不计入误检的例外：广告牌/光滑表面倒影中的人脸或号牌、动物面部（按对象标错才算）、电动自行车/摩托车号牌/临时号牌、喷涂放大号牌等（详见 C.1/C.4）。
- **检出数** N_d：被标记为该类匿名化对象的**总数**。

```
人脸误检率 R_Ff = N_Ff / N_df
号牌误检率 R_Fv = N_Fv / N_dv
```
- **通过：人脸、号牌误检率均 ≤ 10%。**
- 注：需主机厂提供"匿名化区域范围文件"（B.1.3）才能算。

### 指标D · 匿名化效果（§5.6.2.3，附录B.6）

对所有"正检"的匿名化目标，验证**是否仍能被识别**：
- **机器识别（B.6.1）**：2 种人脸识别模型 + 2 种字符/数字/文字识别模型（如 CRNN），看是否还能识别出人脸 / 读出完整号牌。
- **人工识别（B.6.2）**：随机 100 张已匿名化图，人工判断能否确定双眼+鼻+嘴轮廓 / 读全号牌。
- **通过：满足 §5.6.2.1 且已打码的目标应无法被识别。**
- 对应本项目"裁判"核心：打码图上若仍能识别人脸/号牌 → 判不合规。脚本二三引擎检测即充当 B.6.1 机器识别。

## 3. 三个"率"的分母对照（防混淆）

| 指标 | 分子 | 分母 | 通过线 | 强制性 |
|---|---|---|---|---|
| 检出率 | 正检数 | 应检数（正检+漏检） | ≥ 90% | 强制 |
| 漏检率 | 漏检数 | 应检数（正检+漏检） | ≤ 10% | 强制（=1−检出率） |
| 误检率 | 误检数 | 检出数（被标记打码总数） | ≤ 10% | （视作强制） |

## 4. 脚本职责划分（已与需求方确认）

- **自检**（标注精度，B.2.4.2 项1~7）= **独立模块**（现脚本五范畴），用 27 / 11–20 / >20 阈值。脚本三不碰自检。
- **脚本三 = 正式检测计算引擎**：吃脚本二对主机厂匿名化数据的检测输出，算检出率 / 漏检率 / 误检率 / 匿名化效果，出合规结论。
